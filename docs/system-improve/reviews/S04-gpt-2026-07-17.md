# S04 offline review — GPT (via lab bus)

ts: 2026-07-17T15:48:12Z

## 1. Verdict

**Request changes.**

The core direction is sound, but the current plan violates the default-behavior principle, ambiguously filters review scope, and does not carry the contract through every live editor. These are narrow fixes; no architectural rewrite is needed.

## 2. Risks and findings

### (A) Must-fix before merge

- **High — Hidden escaped changes:** “Intersect the panel slice” must be removed; soft mode must retain the entire observed S01 delta and separately annotate out-of-scope paths.
- **High — Contract-blind synthesis:** [grok_synthesis](src/nexus/critique_panel.py:575) is a second live code editor but currently would not receive the contract; it can create files outside the idea scope that cycle-scoped publish may still ship.
- **Medium — Default behavior changes:** `scope_contract_enable=true` violates PRINCIPLES §3; absent/false must preserve the exact existing prompt and review-pack behavior.
- **Medium — Unsafe path semantics:** Define repo-relative, component-aware matching; reject absolute paths, `..`, empty/root prefixes, drive paths and ambiguous backslashes, with forbidden rules always winning.
- **Medium — Fail-open ambiguity:** Fail-open must mean “continue through the legacy path and report non-compliance,” never “use an allow-all contract,” silently omit the warning, or widen publish.
- **Medium — Incomplete flag wiring:** The planned files omit [alive.py](src/nexus/alive.py:40), but the alive.json flag requires an `AliveConfig` field, `from_dict` handling, and `cycle_once` wiring.
- **Medium — Backward compatibility:** New arguments to [implement_portfolio](src/nexus/idea_portfolio.py:711), panel functions, and `write_review_pack` must be optional with legacy defaults.
- **Medium — Untrusted prompt content:** GitHub/arXiv title and concrete text are untrusted; bound and escape them so they cannot terminate the contract delimiter or consume an unbounded prompt.
- **Medium — Schema remains underspecified:** Resolve the “optionally plugins” default, exact field types, and supported `success_check` shapes; do not emit the literal `tests/test_<module>.py`.
- **Low — Ordering claim is inaccurate:** [grok_hard_improve](src/nexus/grok_worker.py:328) adds its own worker preamble, so the contract is first in the idea goal, not D*=0 of the complete model prompt.

The contract should be built once per idea and reused unchanged in the initial Grok goal, critic context, synthesis goal, `CONTRACT.json`, and a nested `MANIFEST.json` field. The `improve_ours` fallback should either receive it through one small optional API or be explicitly recorded as `contract_injected: false`; it must not silently count as contract-covered.

### (B) Nice-to-have

- Use one opt-in `scope_contract_enable=false` flag in v1; enabled always means advisory generation, propagation, persistence, and warnings.
- Write `CONTRACT.json` atomically and store a digest in the manifest to detect prompt/file drift.
- Defer `max_files` and `max_new_files` until counting semantics cover initial implementation and synthesis, or clearly label them advisory.
- Keep GitHub defaults least-privilege; widening to `plugins/` or `skillpacks/` should be deterministic and locally controlled.

### (C) Out of scope

- Contract-based publish filtering, staging changes, or hard enforcement.
- The future `scope_contract_hard_enforce` flag.
- Ledger changes, portfolio-selection changes, and the undefined cycle wrapper.
- Executing `success_check`, SARSI signing/promotion, foundry work, or rebuilding Bubbles.
- Repairing S01’s inability to observe edits to files already dirty before the idea; document that limitation instead.
- A separate existing security issue should be tracked independently: critique-pack path construction accepts raw `cycle_id`, and dot-only slice IDs are not fully excluded. Do not expand S04 to hide this finding, but ensure contract values never participate in path construction.

## 4. Simpler alternative

Use one disabled-by-default flag. Build one pure contract per idea, prepend it to the direct Grok idea goal, pass the identical block to synthesis and critic context, persist it in the pack, and report scope violations against both the initial and synthesis deltas without filtering anything.

Do not add publish behavior, hard flags, ledger fields, cycle wrappers, or success-check execution.

## 6. Bubbles/DNA analogy

**Yes** — it correctly ports the mandatory-envelope discipline to an idea/edit unit rather than rebuilding the Bubbles runtime, but “D*=0/every editor” must be narrowed until synthesis and fallback coverage are accurate.

## 7. Live REAL-loop impact

The live loop can break or become unsafe if:

- Default-on injection changes Grok behavior immediately after the next restart.
- Half-landed signature changes cause `TypeError` during portfolio or panel execution.
- A helper/import/serialization failure aborts an idea instead of taking the legacy path.
- Synthesis edits without the contract and creates globally publishable, out-of-contract files.
- Soft “intersection” hides escaped files from critique or tests.
- Pack failure is swallowed while the original edit remains eligible for later publish.
- A running process imports a partially landed new module.

Keep S02 cycle-scoped publish unchanged, fail open but loudly, and document that out-of-contract cycle-new files remain a residual publish risk until a separate flag-gated publish step.
