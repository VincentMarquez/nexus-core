# S04 Codex offline review — 2026-07-17

Review scope: S04's planned idea scope contract, its supporting principles and Bubbles/SARSI references, and the current integration points in `idea_portfolio.py`, `critique_panel.py`, `alive.py`, `repo_mine.py`, and `publish.py`.

No product code was changed and no tests or REAL cycle were run for this review.

## 1. Verdict

**Request changes.**

The proposed direction is correct: S04 is a small port of the Bubbles/DNA discipline into a per-idea prompt and review artifact. The plan is not merge-ready because its current defaults and “intersect” wording can change live behavior, conceal escaped edits, and imply enforcement that S04 does not provide.

## 2. Risks

### (A) Must-fix before merge

| Severity | Risk |
|---|---|
| high | “Intersect” must not filter the S01 strict slice: doing so can hide escaped edits from the review diff, focused tests, snapshot/restore, and later publish handling while leaving the files changed. |
| high | A naïve prefix check permits path escape and prefix collisions; paths must be lexically normalized, segment-aware, repo-relative, and deny-first. |
| high | Idea IDs, titles, and missions are untrusted research data; placing them first in a D*=0 block without an encoding boundary lets prompt text forge fields or close the DNA block. |
| medium | `scope_contract_enable=true` changes live prompts by default, contrary to the opt-in rule in `PRINCIPLES.md`. |
| medium | The contract is not currently carried into critique-panel Grok synthesis or the `step_improve_ours` fallback, so the plan's “every implement unit” claim is false unless those paths are covered or explicitly excluded. |
| medium | Adding config fields or required function parameters without wiring `AliveConfig.from_dict` and every call site can stop the REAL loop before an idea reaches its worker. |
| medium | Contract build, format, or artifact-write errors can become a fail-open blind spot or abort the portfolio; observation-mode fallback must preserve the legacy path and record a structured error. |
| medium | `success_check`, `max_files`, and `max_new_files` are not enforced by S04; presenting them as satisfied would confuse “worker returned” with acceptance. |
| medium | `owner: alive_real` conflates the executor/provenance identity with external authority; use code-owned `created_by` for the executor and a trusted operator value for `owner`. |
| medium | A dirty path-set delta cannot see further edits to a path already dirty before the idea; advisory results may say only “no observed violation” and must expose incomplete coverage. |
| low | An unbounded DNA block increases prompt size and can crowd out the task or produce nondeterministic behavior. |
| low | The block can be first in the task payload, but it is not literal final-prompt D*=0 because `grok_hard_improve` prepends its worker wrapper. |

Required resolutions:

1. Make S04 opt-in with one flag, `scope_contract_enable=false`, and wire it through `AliveConfig`, `from_dict`, and the `implement_portfolio` call. With the flag off, prompt construction and artifacts must follow the exact prior path.
2. Replace “intersect the slice” with “classify the complete strict slice and record warnings.” Never remove out-of-scope files from the diff, tests, snapshot/restore set, or publish inputs in S04.
3. Treat allowed/forbidden prefixes, schema, and limits as code-owned policy. `created_by` records `alive_real`; `owner` comes from trusted operator context or a fixed external-operator sentinel. An idea may supply mission data, but it may not grant itself authority or broader scope.
4. Define path semantics before implementation: convert to a canonical repo-relative POSIX form; reject absolute paths, drive-qualified paths, backslashes, NULs, empty/root paths, and any `..`; compare full path components; apply forbidden prefixes before allowed prefixes. This classifier is advisory in S04, not a filesystem sandbox.
5. Serialize untrusted idea fields as bounded data inside a fixed policy envelope. A title or mission containing newlines, JSON/XML delimiters, or a closing tag must not create a new instruction or contract field.
6. State the delivery boundary precisely. The same per-idea contract should reach each editor claimed by S04, including panel synthesis. If the fallback worker is intentionally not changed in v1, record it as unsupported and narrow the claim instead of implying complete coverage.
7. Keep `write_review_pack(..., contract=None)` and any new worker arguments optional and keyword-only. Guard contract construction and writing so observation failure falls back to the old prompt, while emitting a visible `scope_contract_error`.
8. Mark `success_check` as `not_evaluated` metadata. File-count limits must either produce warnings now or be explicitly reserved for a later acceptance/enforcement step.
9. Do not add an unused `hard_enforce` flag. A dormant security-looking switch creates fail-open false assurance; hard gating belongs to the later Accept/quarantine step.

### (B) Nice-to-have

- Inject a clock into contract construction so `created_ts` is deterministic in tests.
- Keep the DNA block compact, with documented line and byte limits.
- Put a schema/path/digest reference in `MANIFEST.json` so duplicated contract values cannot drift from `CONTRACT.json`.
- Use stable warning codes such as `path_out_of_scope`, `contract_build_error`, and `contract_delivery_unsupported` in addition to human-readable text.
- Start with one restrictive, code-owned prefix table; defer source-specific policy tables until evidence shows they are needed.

### (C) Out of scope

- Hard blocking, quarantine, and acceptance decisions (S05/S06).
- Any `publish.py` staging change or contract/publish intersection.
- S09's worktree/path jail.
- Executing `success_check`, changing `entry["ok"]`, or implementing held-out evaluation.
- A cycle-wide wrapper, global DNA injection into unrelated workers, or converting every existing Grok call in this step.
- Rebuilding UniversalBubble, the Bubbles bus, D* propagation, a SARSI foundry, signing, or weight training.
- Repairing existing dirty-worktree and pre-staged-file limitations. S04 must not claim that a prompt contract closes those independent publish risks.

## 3. Missing tests

Add focused coverage for:

1. `AliveConfig` defaults the flag to `false`, preserves explicit `true`/`false` through `from_dict`, and passes it to `implement_portfolio`.
2. Disabled mode produces the exact legacy direct-Grok goal, creates no contract artifact, and leaves existing callers valid.
3. Enabled mode places the fixed contract policy first in the initial editor prompt and delivers the same contract to panel synthesis; fallback delivery is either tested or explicitly reported unsupported.
4. Sparse and malformed ideas still produce a deterministic, bounded contract; `created_by` identifies the executor, `owner` comes from trusted operator context, and idea fields cannot override either field, schema, allowed/forbidden prefixes, or limits.
5. Path classification covers valid descendants plus absolute paths, `..`, empty/root paths, NULs, drive paths, backslashes, `src/nexus_evil`-style prefix collisions, and forbidden-over-allowed precedence.
6. Adversarial IDs, titles, and missions containing newlines and closing delimiters cannot escape the untrusted-data field or forge DNA policy.
7. Soft mode reports escaped paths but keeps the complete S01 strict slice in the diff, tests, snapshot, restore, and result metadata; a pre-dirty baseline or observation error sets `coverage_complete=false` and never reports verified compliance.
8. Existing `write_review_pack(...)` callers work with no contract; enabled calls write valid `CONTRACT.json`, and its recorded contract matches the prompt contract.
9. Contract builder, formatter, and artifact-write exceptions do not abort the remaining portfolio; the legacy path runs and a structured error remains observable.
10. `max_files` and `max_new_files` boundary behavior is warning-only, or those fields are absent/reserved. `success_check` is never executed and cannot change implementation success in S04.
11. Two ideas in one portfolio receive isolated contracts and review packs with no cross-idea leakage.
12. Run the new scope-contract tests, relevant critique-panel and Alive config/pass-through tests, existing publish regressions, and the import smoke for `alive`, `idea_portfolio`, `critique_panel`, and `scope_contract`.

The current test plan names only `test_scope_contract.py` and an import check; that is insufficient for changes spanning live configuration, prompt delivery, and review-pack behavior.

## 4. Simpler alternative

Use one pure `scope_contract.py` helper and one default-off flag.

When enabled, it should:

1. Build a compact canonical contract from fixed policy plus bounded idea data.
2. Prepend that contract to the S04-covered editor prompts.
3. Write `CONTRACT.json` alongside the existing review pack through an optional argument.
4. Classify the unchanged S01 slice and record warning-only `in_scope`/`out_of_scope` results.

Defer source-specific policies, the cycle wrapper, hard flags, publish changes, and success evaluation. This provides the intended discipline without changing the operational spine.

## 6. Bubbles/DNA analogy

**Yes.** S04 correctly ports the discipline of mandatory identity, context, and bounds carried with a work unit; the block is first in the task payload, not literal final-prompt D*=0, because `grok_hard_improve` prepends its worker wrapper.

The wording must still distinguish the mechanisms: Bubbles was constructor-level and fail-closed, while S04 is a default-off, warning-only prompt/artifact contract. Narrow “every implement unit” to the editors actually wired and tested in S04.

## 7. Anything that could break the live REAL self-improve loop

- Default-on injection would change the next REAL process without an explicit rollout decision.
- A missing `AliveConfig.from_dict` field, a new required argument, or an omitted new module in the commit can break startup or the `alive → implement_portfolio → critique_panel` chain.
- Contract construction currently belongs near code that runs before the worker's per-idea exception boundary; an unguarded exception there can abort more than one idea.
- Filtering the strict slice can hide files that remain modified, so they escape critique/restoration and may later enter a publish set.
- Injecting the contract into the initial prompt but not panel synthesis or fallback gives false assurance while a later editor remains unconstrained.
- Unbounded or injectable D*=0 content can materially change the worker's behavior and prompt cost.
- Silently falling back on validation errors makes telemetry claim coverage that did not occur. Observation mode may fail open only if the old behavior is preserved and the failure is recorded.
- S04 must take effect only on the next process. Do not hot-patch or restart a running REAL cycle as part of this step.
- **Existing publish P0, separate step—not S04:** a failed baseline `git status` can collapse to an empty baseline and broaden staging to all allowlisted dirty paths.
- **Existing publish P0, separate step—not S04:** plain `git commit` can include unrelated files that were already staged whenever the cycle stages at least one new path.

At review time no matching NEXUS `alive` process was observed. That is a point-in-time check, not authorization to start, stop, or restart the live loop.
